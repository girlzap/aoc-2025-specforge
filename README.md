## ğŸ§± What is Spec-Driven Development?

Spec-Driven Development is a workflow where you:

1. **Start with requirements** (e.g., the Advent of Code puzzle text).
2. **Turn them into a formal spec**, including:

   * problem summary
   * domain entities
   * inputs + constraints
   * expected outputs
   * rules & logic
   * edge cases
   * sample tests
   * solution outline
   * flow walkthrough
   * implementation plan
3. **Write code only after the spec is complete.**

Itâ€™s like a lightweight, human-readable version of TDD or BDD.
